# AI Decoded - An AI Encyclopedia - 2026

Built for people who want to actually understand AI and it’s many types and terminologies. This is a standalone HTML file with 145 terms, five study modes, there’s no login, no account needed, no backend BS to deal with. You just open it and dive into learning.

This project is a work in progress.

## What's in this file

**Learn** covers the actual concepts of how models work, what the different types of AI are, what tools exist and what they're good for, how to prompt well, and what the ethical concerns are worth taking seriously. Each topic opens into its own detailed view with expandable sections.

**Flashcards** lets you study by category or run through all 145 terms at once. You mark each card as "Got It" or "Still Learning" and the deck tracks your progress with a progress bar per category.

**Quiz** pulls random questions from the full library in three formats: define the term, name the term from its definition, or identify which category it belongs to. It keeps a running score for the session.

**Glossary** is the searchable reference view. Filter by category or search by keyword and click any term to expand the definition.

**Challenge** is the hard mode. A term appears and you write the definition from memory. It scores your answer against the actual definition using keyword matching and tracks your streak.

## Progress saving

Progress saves automatically to your browser's local storage. It remembers which terms you've marked as known or still learning, your quiz scores, your streak, and whether you had dark mode on.

One thing worth knowing: local storage only works when the file is served from a real URL. If you're opening it by double-clicking the file on your desktop, some browsers will block storage entirely and nothing will save. I am hosting this on GitHub Pages and it works the way it's meant to.

## Changing things

Everything is in one file. The term list is a JavaScript array starting around line 518, each entry is just a term, definition, and category. The styles are CSS custom properties at the top of the file and cover both light and dark mode. Nothing requires a build step or a package manager.
